Replacing Text

Yes, we have the technology! Any text you see in the template, you can replace! Be careful though, the replacement function we have is kind of brutish: It replaces every occurence of the query text within a particular slide. The function is here:

  • powerpoint.replaceText(query_text, new_text, slide_index)
    Replaces all occurences of the query string query_text with the string new_text within the slide specified by the integer slide_index.

If you look at the slides, you can see that all four templates have a textbox with "<metadata>" in them. We're gonna do some exercises just to see how this function works in action.

  1. What will happen if I run powerpoint.replaceText("<metadata>", "New York City", 1) on the initial PowerPoint document?
  2. Write the script that "removes" the "<metadata>" text from all the slides.
  3. Write the script that replaces "Slide i" in place of "<metadata>", where i changes dynamically based on the index of the slide we're in.

Populating Images

I know what you're thinking. "Hey! So sure, we can rearrange slides, and replace texts... what if I want some images to go with that??" Don't you worry, we gotchu covered.

powerpoint.addImage(
image_obj,
slide_index,
width_percent,
height_percent,
x_offset_percent,
y_offset_percent,
options_obj
);

Going through each of the parameters:

  • image_obj
    An Image Object, which can be created with the syntax: new Image(url), where the url is that of the image we want to populate.
  • slide_index
    Integer or Object. If an integer, represents the index of the slide we want to populate the image into.
    If object, needs to use the following syntax: where(0).slides(slide_index).shapes(0)
    An image can be populated into a particular layer if the object is used instead of an integer index.
  • width_percent
    A String, with the format "xxx%", which represents the width of the image, where the unit is the percentage of the slide width.

    As an example: setting this to "50%" would make the image half as wide as the PowerPoint slide's width.
  • height_percent
    A String, in the same format as width_percent, but indicating image height.

    Example: setting this to "50%" would make the image half as tall as the PowerPoint slide's height.
  • x_offset_percent
    A string, with the format "xxx%", which indicates where the top left corner of the image should be placed. Unit is the same as width_percent: the percent is the percentage of the PowerPoint slide width.

    Example: setting this to "50%" would make the image's left side start at the middle of the slide (horizontally).
  • y_offset_percent
    A string, in the same format as x_offset_percent.

    Example: setting this to "50%" would make the image's top side start at the middle of the slide (vertically).
  • options_obj
    JS object ({}). Can contain attributes to be used, most notably crop.

Okay that's actually a lot of info. For now the most relevant parameters you need to use for now is image_obj and slide_index.
