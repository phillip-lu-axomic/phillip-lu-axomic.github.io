Grabbing Metadata
What kind of info are we even grabbing?
In the next few sections, we will learn about how to grab metadata, for use in populating the templates.

Projects

Projects have three categories of metadata:
  • Display Fields
    These are built-in fields, and includes "Project Name" and "Project Code".
  • Project Fields
    These are custom fields, and include metadata that aren't usually repeated over multiple projects. (i.e. Project Cost, Start Date, etc.)
  • Project Keywords
    These are custom keywords, and include metadata that are usually repeated over multiple projects. (i.e. Country, Project Type, etc.)

Files/Albums

Files have two file-specific categories of metadata, as well as accessors to access project-based metadata.

  • File Display Fields
    These are built-in fields for files. For the most part we won't use these, as these contain the OpenAsset file names, and we don't use them often.
  • File Fields
    These are custom fields for files, and include file-specific metadata.
  • File Keywords
    These are custom keywords for files.
    NOTE: WE CANNOT GRAB FILE KEYWORDS FOR PROJECT/ALBUM-BASED TEMPLATES!!!!
    The CSCs are still learning this fact. If a CSC asks you to grab file-based keywords: WE CAN'T DO IT FOR PROJECT/ALBUM-BASED TEMPLATES!!!
