Just navigate over to the OpenAsset instance, and go to the following URL:

https://[your client's instance].openasset.com/Page/Scripts
You should see a list of templates available. If you'd like to create a new template, just click on the "Create New Template" button. You should see the following prompt:

  • Name:
    Indicates the name of the template. Clients will see this!
  • Application:
    What kind of document are we exporting? InDesign/PowerPoint/Word.
  • Language:
    What language we're writing the script in. Only "Awesome" applies: The others are phased out now.
  • Module:
    What type of datatype is the client selecting? Project/Album/Employee

"Application" will be explored in depth in the "Populating Templates" section of the tutorial.
"Module" will be explored in depth in the next section ("Grabbing Metadata")

Go ahead and create a Project-based, InDesign template in your demo instance. You should see the template entry on the bottom of the Script page.
Clicking on the "Edit" icon will show the main scripting page:

This is where we script out the client's templates. On the left side you'll find the script metadata, which you can edit to your heart's content.

Beneath the script metadata, we have the "Script Data JSON" section. This is the JSON that will be sent to the backend whenever someone wants to generate a template.
This will be discussed in greater depth in each "Grabbing Metadata" section.

At the top of the middle column, we have a place for uploading the base template.
Below that is an "Example Data Object". This is where you can search for a Project or Album, and display what the variable "data" looks like for that particular object.

And finally, to the right, we have the actual script that will get run anytime someone wants to create this particular template.
This is where you'll be writing the script that will grab metadata, and populate them into the template document.

Don't worry if this is too much. There will be a few exercises in each section to reinforce the lessons in each section.
